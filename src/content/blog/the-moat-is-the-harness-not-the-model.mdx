## Opening Thesis

- The dominant AI narrative — "build the smartest agent, wire it to the best model, win" — is wrong
- **The model layer is already commoditized**: GPT-4o, Claude Opus, Gemini 2.5, DeepSeek cluster within single-digit percentage points on benchmarks
- Raw intelligence is converging. Inference costs are collapsing. Open-source catches up in months
- **Two things compound together** to create durable advantage:
  - The **agentic harness** — orchestration, guardrails, memory, tool-use, workflow integration
  - The **data flywheel** — the proprietary feedback loop that makes your system smarter with every use
- The LLM is becoming the CPU. The question is: what becomes the operating system?

## Why the Harness Creates the Moat, Not the Model

- **Berkeley BAIR (Zaharia et al.)**: "State-of-the-art AI results are increasingly from compound systems, not monolithic models"
  - Systems beat models when: design > scaling, dynamism needed, control/trust required, goals vary
- **Karpathy**: LLM = kernel in a larger OS; tool access, memory, I/O, code execution = the real product

### What Exactly Is the Agentic Harness?

- **Tool integration** — browsers, terminals, APIs, file systems
- **Context & state management** — memory, session persistence
- **Evaluation frameworks** — continuous output verification against business rules
- **Observability pipelines** — hallucination, drift, failure monitoring
- **Security guardrails** — deterministic validation of probabilistic outputs

## The Data Flywheel: Your Only Durable Advantage

- General internet-scale data is abundant and shared
- What moves the needle: **specialized, high-signal data generated in real time inside closed-loop systems**

### Flywheel Building Blocks

- **HITL workflows** — gold-standard data no scraper can replicate
- **Workflow integration** — AI inside EHR/CRM/checkout = you own the context
- **Low-latency feedback** — faster learning = faster competitive separation
- **Proprietary generation** — data that can only exist through your product

### TCO Reduction Through the Flywheel

- Start with expensive 70B+ parameter models for complex tasks
- Collect successful interactions → curate ground-truth dataset → fine-tune smaller models (1B–8B)
- **A customized 1B model on proprietary data achieves ~96% accuracy of a 70B model** on the same task

<Callout type="info" title="The Modern Data Moat Has Evolved">
Static data moats are vulnerable — LLMs can generate synthetic approximations. True defensibility comes from owning the software interfaces where data is **naturally and continuously generated**.
</Callout>

## Why Vertical SaaS Is Still a Play

### 1. The Data Is the Moat, Not the Feature

- General AI legal assistant: ~15% churn (too broad)
- Vertical SaaS for medical malpractice discovery: **300% higher pricing, less than 3% churn**

### 2. Workflow Integration Creates Structural Lock-in

- Leaving Procore = ripping out field coordination, compliance docs, bid management
- The output is a byproduct of deep integration

### 3. Embedded Financial Infrastructure

- Toast, ServiceTitan: process payments, payroll, merchant accounts
- LLMs can't settle payments, underwrite loans, or interact with banking infrastructure
- Revenue via **transaction volume** — immune to seat compression

### 4. The Numbers

- 2025: verticals = **53% of deal volume**, **30% of capital** ($186B)
- Excluding mega-rounds: **51% of total capital**
- 2026: AI-native vertical SaaS seeing **65% increase** in capital flow

### 5. Historical Parallel

- Despite horizontal cloud dominance → Veeva (~$37B peak), Toast, Procore, ServiceTitan
- **Top 20 public vertical SaaS: ~$300B combined market cap**
- CrowdStrike CEO: *"Hyperscalers providing all security services? That didn't happen."*

## Horizontal AI vs. Vertical SaaS

| Dimension | Horizontal AI | Vertical SaaS |
|---|---|---|
| **Data Ownership** | Public/purchased data | Proprietary, industry-specific |
| **Workflow Integration** | Surface-level, general-purpose | Deeply embedded in domain workflows |
| **Competitive Moat** | Model performance + brand | Data flywheels + workflow + domain expertise |
| **Revenue Model** | API licensing | SaaS subscriptions + embedded finance |
| **Switching Costs** | Low — API swaps trivial | High — ripping out embedded workflows |
| **Regulatory Depth** | Broad (HIPAA, SOC2) | Deep, jurisdiction-specific |

## The Playbook: Building for the Long Game

<Callout type="success" title="Five Principles for Durable AI Advantage">

1. **Own the workflow, not the output** — embed where switching costs are prohibitive
2. **Build the harness first** — orchestration, guardrails, memory, observability; model-agnostic
3. **Spin the flywheel from day one** — every interaction = proprietary data; HITL + eval loops
4. **Compete on integration, not intelligence** — let model labs race on benchmarks; your advantage is domain depth
5. **Price for outcomes, not seats** — value is in the labor line of P&L being replaced

</Callout>

## Cognitive Debt: The New Technical Debt

- **Cognitive debt** = AI-era successor to technical debt
- Manifests when enterprises deploy agents without a robust harness:
  - Chronic context loss across interactions
  - Uncontrolled and hallucinated tool usage
  - Unreliable agent behavior
- Randall Hunt: *"If you don't engineer the harness, you don't get compounding leverage; you get compounding cognitive debt."*
- Also impacts **human capital**: junior employees deskilled by unharnessed AI; a robust harness forces AI to show work, cite sources, pause for HITL verification

## Conclusion

- Model layer follows classic commoditization: initial scarcity → rapid proliferation → price collapse → value migration
- Platforms entering verticals face the same old constraint: **they cannot go deep enough** into domain-specific data loops, regulatory workflows, and integration layers
- The real competition is in the harness and the data flywheel spinning inside it — that's where durable value gets built

<Callout type="insight">
The best AI companies won't have the best models. They'll have systems that get smarter every day — because the harness captures what the model can't.
